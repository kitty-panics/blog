使用 Python3 解压乱码的 zip 文件

Kitty • Panics  included in Python3
 2021-05-07  259 words   One minute 
Contents
  • 代码

由于 zip 压缩格式默认没有指定编码格式，所以会使用系统默认的编码 (在简中 Windows 上通常是 GBK，在 Linux 上通常是 UTF-8)，这会导致在 Linux 上解压 Windows 中创建的 zip 包会乱码。

虽然这个问题有很多种解决方案，但个人比较喜欢用 @Latm Ake 写的 Python2 脚本。

不过 Python2 已于 2020-01-01 EOL 了，现在是时侯弃用了。所以将其改一改，迁移至 Python3 版本。

# -*- coding: UTF-8 -*-

# 导入模块
import os
import sys
import zipfile

# 打印和读取需解压的文件
print("=> Processing File: " + sys.argv[1])
GET_ZIP_FILE = zipfile.ZipFile(sys.argv[1], 'r')

# 循环处理文件列表
for GET_NAME_LIST in GET_ZIP_FILE.namelist():
    try:
        SET_NAME_LIST_UTF8 = GET_NAME_LIST.encode('CP437').decode('GBK')
    except UnicodeEncodeError:
        SET_NAME_LIST_UTF8 = GET_NAME_LIST.encode('UTF-8').decode('UTF-8')
    print("- Extracting: " + SET_NAME_LIST_UTF8)

    DIR_PATH = os.path.dirname(SET_NAME_LIST_UTF8)
    if not os.path.exists(DIR_PATH) and DIR_PATH != '':
        os.makedirs(DIR_PATH)

    GET_FILE_DATA = GET_ZIP_FILE.read(GET_NAME_LIST)
    if not os.path.exists(SET_NAME_LIST_UTF8):
        with open(SET_NAME_LIST_UTF8, 'wb') as FILE_OPEN:
            FILE_OPEN.write(GET_FILE_DATA)
GET_ZIP_FILE.close()
