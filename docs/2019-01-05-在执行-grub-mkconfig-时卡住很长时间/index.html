在执行 grub-mkconfig 时卡住很长时间

Kitty • Panics  included in GRUB2
 2019-01-05  127 words   One minute 
Contents
  • Error
  • Fix

Error

最近需要把系统重装在 LVM on LUKS 上，在安装的最后一步 grub-mkconfig 卡住了，提示以下错误。

1

WARNING: Device /dev/xxx not initialized in udev database even after waiting 10000000 microseconds

Fix

这个错误是由于 lvm2 上游的变动，导致 GRUB2 出现了问题。有两种解决方案。

  1. 通过 mount --bind 让 chroot 中能访问到 /run/lvm 来解决。
    1. mkdir /mnt/hostlvm
    2. mount --bind /run/lvm /mnt/hostlvm
    3. arch-chroot /mnt
    4. ln -s /hostlvm /run/lvm
  2. 通过 pacman -U 命令安装 旧版本 的 lvm2 来解决。
