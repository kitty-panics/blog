proxychains-ng 对 Chromium 无效

Kitty • Panics  included in 软件
 2022-10-11  277 words   One minute 
Contents
  • Error
  • Fix

最近 Google 关闭国内服务器的翻译 API 服务，才发现 Chrome 自带的网页翻译居然不受扩展控制， 尽管设置中显示 “Chrome is using proxy settings from an extension”。

那些改 Hosts 的方法就懒的折腾了，直接代理 Chrome 所有流量更简单。当然这里的全局代理并不是 TPROXY 之类，而是用 proxychains-ng 实现对单个应用的全局代理。

Error

但在实际使用时，发现对 Chromium (Electron) 系应用无效，而且并没有任何错误信息。翻了下 Commits，作者在 2020 年才解决这个问题。

  • experimental new feature: proxy_dns_daemon · rofl0r/proxychains-ng@7fe8139

Fix

  1. 注释配置中的 method 1 并反注释 method 3 (也就是 proxy_dns_daemon)。
  2. 新建一个 systemd service，并启用，内容如下：

[Unit]
Description=DNS daemon for proxychains4
After=network-online.target nss-lookup.target
Wants=network-online.target

[Service]
CapabilityBoundingSet=CAP_NET_BINDING_SERVICE
AmbientCapabilities=CAP_NET_BINDING_SERVICE
NoNewPrivileges=true
ExecStart=/usr/bin/proxychains4-daemon

[Install]
WantedBy=multi-user.target
