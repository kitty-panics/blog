2021-04-22 日的更新

最近发现一个很棒的多系统启动盘制作工具：Ventoy 。

只需要把 ISO/WIM/IMG/VHD(X)/EFI 等类型的文件拷贝到 U 盘里面就可以启动了，无需其他操作。 还可以一次性拷贝很多个不同类型的镜像文件，Ventoy 会在启动时显示一个菜单来供选择。

 

 

经常折腾系统的人，肯定得一手一个维护盘 (又称 Live 盘，救援盘，启动盘)。 制作维护盘的工具就那几个。

  • Windows
    1. UltraISO (太旧了，已不建议使用)
    2. Rufus
    3. ……
  • Linux
    1. dd
    2. ……

虽然这些工具用起来都很简便，但有个坏处就是会占用整个 U 盘，维护盘虽然不常用， 但关键时候又不可或缺，

如果使用大容量 U 盘而又只用来用作维护盘就太不划算了。这个问题可以通过使用 GRUB2 多重引导来解决。

准备工作

  1. 一个尽可能大的 U 盘
  2. mkfs.fat -F32 这个 U 盘
  3. 在 U 盘上安装 GRUB2

引导 Arch Live CD

编辑 /etc/grub.d/40_custom 文件，在文件末尾添加如下代码。

# Arch Linux Live CD
menuentry "[*] Live CD for Arch Linux" {
    # ROOT 所在的设备 (使用设备名)
    set root="(hd0,gpt1)"
    # ROOT 所在的设备 (使用 UUID)
    set imgdevpath="/dev/disk/by-uuid/XXXX-XXXX"

    # ISO 镜像所在的路径
    set isofile="/ISO/archlinux.iso"
    # 将 ISO 镜像映射为回环设备
    loopback loop $isofile

    # 载入 Linux 内核及其相关参数
    echo "载入 Linux ..."
    linux (loop)/arch/boot/x86_64/vmlinuz-linux quiet loglevel=0 vga=current fbcon=nodefer sysrq_always_enabled=1 img_dev=$imgdevpath img_loop=$isofile earlymodules=loop
    # 载入 Intel 的微码、AMD 的微码、initramfs
    echo "载入初始化内存盘..."
    initrd (loop)/arch/boot/intel-ucode.img (loop)/arch/boot/amd-ucode.img (loop)/arch/boot/x86_64/initramfs-linux.img
}

注：

  1. 需根据自己的情况修改相应的内容 (UUID、内核参数、镜像名等)
  2. 如果 BOOT 和 ROOT 分区使用了 LUKS 加密，建议将 ISO 镜像放在 EFI 分区

引导其它 Linux 的 Live CD

引导其它 Linux 的 Live CD 和 Arch Live CD 大同小异，可参考 thias/glim 项目进行操作。

WePE (微 PE)

虽然日常使用 Arch Linux，但也免不了帮别人修修电脑，所以得有个 PE，目前最放心的 PE 就只剩 微 PE 了。

编辑 /etc/grub.d/40_custom 文件，在文件末尾添加如下配置。

# Windows PE
menuentry "[*] PE for Windows" {
    # ROOT 所在的设备 (使用设备名)
    set root="(hd0,gpt1)"
    # ROOT 所在的设备 (使用 UUID)
    set imgdevpath="/dev/disk/by-uuid/XXXX-XXXX"

    # EFI 文件所在的路径
    set efifile="/EFI/BOOT/WEPEx64.efi"

    # 载入 WePE
    echo "载入 WePE ..."
    chainloader $efifile
}

注：

  1. 需根据自己的情况修改相应的内容 (UUID、efi 文件名等)
  2. 此方法是以 EFI 的方式启动 WePE，如果想以 BIOS 的方法启动，请百度之
