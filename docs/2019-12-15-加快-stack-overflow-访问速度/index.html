2021-11-22 的更新

此方法已失效，因为 Stack OverFlow 现在搜索时需先通过 Google 的 reCAPTCHE 验证码， 禁用掉 JavaScript 也就无法加载验证码了。

reCAPTCHE 验证码脚本共有两个地址，默认使用第一个 (但第一个国内通常无法访问)。

  1. https://www.google.com/recaptcha
  2. https://www.recaptcha.net/recaptcha

但可以用 kitty-panics/ReplaceGoogleCDN 浏览器扩展将其替换成第二个。

 

 

常说面向 Stack OverFlow 编程，但是在国内 Stack OverFlow 加载实在太慢了，严重影响 Ctrl-C、Ctrl-V 效率。

其实 Stack OverFlow 加载慢主要是因为部分 JavaScript 脚本在国内无法正常访问造成的。 不过如果 Stack OverFlow 只是用于查询和浏览的话，那么不需要 JavaScript 也能正常显示内容。 既然如此，则可以将 JavaScript 禁用达到加快访问 Stack OverFlow 的目的。

这一目的可以通过 uBlock Origin 来实现 (其它同类工具也能实现，只是个人喜好罢了)。

禁用 JavaScript

在 自定义动态规则 -> 永久规则 下添加下面内容。

1
2
3
4
5
6
7

no-scripting: askubuntu.com true
no-scripting: mathoverflow.net true
no-scripting: serverfault.com true
no-scripting: stackapps.com true
no-scripting: stackexchange.com true
no-scripting: stackoverflow.com true
no-scripting: superuser.com true

移出影响阅读的内容

禁用 JavaScript 后，页面中会有元素遮挡正文内容，所以也得移除这些元素。在 自定义静态规则 中添加如下内容即可。

1
2

stackoverflow.com##.js-consent-banner.r16.l16.b16.bar-lg.fc-white.bg-black-750.sm\:p16.p32.sm\:w-auto.ws4.z-nav-fixed.ps-fixed.ff-sans
stackoverflow.com##.js-dismissable-hero.z-nav.b0.r0.l0.ps-fixed.bc-black-100.bt.bs-sm.bg-black-025
