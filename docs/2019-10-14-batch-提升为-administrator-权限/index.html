Batch 提升为 Administrator 权限

 2019-10-14  723 words   2 minutes

尽管 Windows 用户都默认使用 Administrator 帐户，但它虽名为 Administrator，其实并没有 Administrator 全部的权限，很多操作会受到限制。

所以在执行某些 Batch 脚本时需要提升权限，每次右键 以管理员身份运行 (A) 太麻烦了。 可在脚本开头插入下面这一行代码。

代码

1

%1 start "" mshta VBScript:CreateObject("Shell.Application").ShellExecute("cmd","/c pushd ""%~dp0"" && ""%~s0"" ::","","runas",1)(window.close) && exit
  • %1: 参数 1 是占位符，开始运行时为空会被忽略掉。后面将传入参数 ::，表示注释掉此行。
  • start "": 启动一个程序，"" 表示窗口标题为空。
  • mshta VBScript:: 启动的程序是 mshta，将用它运行一段 VBScript 代码。
  • CreateObject("Shell.Application").ShellExecute(): 使用 Shell.Application 对象的 ShellExecute 方法来执行一个命令。
    • cmd: 执行的命令是 cmd，再通过 cmd 执行两条命令。
    • /c pushd "%~dp0": 先通过 pushd 命令改变工作目录为脚本所在目录。
    • && "%~s0" ::: 前者执行成功后，就执行脚本自身。并传入参数 ::。
    • "": 这个空参数本来用于设置工作目录，但某些情况会无效，所以置空不用。
    • "runas": 以管理员身份执行命令。
    • 1: 执行命令的窗口保持原样。
  • (window.close): 关闭打开的窗口。
  • && exit: 前一个命令执行成功，就退出当前脚本。

隐藏窗口

如果你想在提升权限的同时，隐藏 CMD 窗口。可将上面的 1 改为 0。

  • 0: 隐藏窗口。这将使脚本在后台运行，用户将看不到脚本运行窗口。
  • 1: 正常状态显示窗口，通常会显示脚本运行窗口。
  • 2: 最小化状态显示窗口。这将最小化脚本运行窗口。
  • 3: 最大化状态显示窗口。这将最大化脚本运行窗口。

参数 1 被占用

上述代码唯一的缺点就是参数 1 会被占用，在新脚本中使用还无所谓，放进旧脚本还得重新排列参数， 稍不注意就出错了。所以稍加改良。

1

fltmc >nul 2>&1 || start "" mshta VBScript:CreateObject("Shell.Application").ShellExecute("cmd","/c pushd ""%~dp0"" && ""%~s0""","","runas",1)(window.close) && exit

fltmc >nul 2>&1 命令需要管理员才能执行，所以开始时会失败，失败后就执行后面的提权命令。

扩展阅读

Administrator 用户并不是系统中最高权限，在它上面还有 SYSTEM、NT SERVICE\TrustedInstaller。

如果想获取最高权限，可以使用 NSudo 这个工具。
