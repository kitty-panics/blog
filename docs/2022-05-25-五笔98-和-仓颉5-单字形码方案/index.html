2023-07-14 的更新

重写了静态全码重码统计工具，列出了支持 Unicode CJK 大字符集形码的重码数据，见 kitty-panics/DCoSC 。

 

 

因想要完美而开始的项目，终会因不完美而结束。

最初受不了拼音的码长太长、重码太多，开始使用双拼 (经历：自然码 => 微软双拼 => 小鹤双拼 )。 虽然双拼每字两码很有节奏感，但并没有解决重码问题。后面发现小鹤双拼还有个配套的音形码， 就入了音形码的坑。

然后发现小鹤音形词库太大 (说它大吧，又经常没有我要的词)，反而造成了大量的重码，且又不公开码表 (虽然可以解码码表)。最终决定设计一款低重码的 单字音形码方案 (见 《个人自制的输入法方案》 )。

放弃旧方案

自制的单字音形码方案使用了两年，还是决定放弃，选用 “五笔98” 和 “仓颉5” 重新做了一个 单字形码方案， 收字范围分别为 GB18030-2000 ，有以下原因。

  1. 一年前硬盘炸过一次 (见 《重装 Arch Linux 遇到的新坑》 )，原始数据丢了，重新做太麻烦。
  2. 音形码方案有个弊端，遇到 “知道咋写但不知道咋读” 和 “知道咋读但不知道咋写” 的字，都打不出来。
    • 还有数以万计的汉字压根就没有拼音。
  3. 其中的拼音数据，去查字典吧，几万汉字，工作量太大。爬取字典网站吧，错误又多，勘误还困难。
    • 严格来说也不算错误拼音，而是以前的标准读音，只是后面标准变了。但保留的话会造成更多的重码。
  4. 音托类字根虽然减少了记忆量，但也会在脑中多一步转换操作，通过这两年的使用，发现音托并不适合我。
  5. 不确定是不是因为音形码的原因，越来越多的字不会写了。

筛选新方案

虽然已经过了那个 万码奔腾 的时代，但也有新的形码方案时不时冒出来。至于为什么选择 “五笔98” 和 “仓颉5”？

  • 分别是简中和繁中下普及率最高的形码。
    • 使用他人电脑时，随便安装个输入法就能打字了 (可惜郑码一手好牌，打得稀烂)。
  • 社区爱好者在维护时选择了 Github 这个平台，下载更新、反馈和多人协作都非常的方便。
    • 其它绝大部分形码就只有个 QQ 群，群文件都是乱七八糟的。
    • 少部分有个网盘，网盘要不很久没有更新，要不链接已失效。
  • 用户基数大，就算官方不维护了，也有大量高水平的志愿者来维护，能坐享其成。
    • 其它形码原作者不维护了就真的没了。

当然现在也有一些评测数据很棒的形码 (比如虎码……)，但对比下来也没有谁能比大家高出一个量级， 只能说 诸家修为，各有所长，那就 青菜萝卜，各有所爱 呗。
