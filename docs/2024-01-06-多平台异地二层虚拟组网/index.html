多平台异地二层虚拟组网

Kitty • Panics  included in 软件
 2024-01-06  521 words   2 minutes 
Contents
  • 选择
  • 安装

去年有篇文章介绍了使用 KDE Connect 进行多平台粘贴板同步，跨局域网通信用 Tailscale 、 ZeroTier 之类的软件。

  • 多平台轻量化粘贴板同步方案 - Blog of Kitty • Panics - 知知为知知

这类软件虽然很方便，但有时并不能打洞成功，走中转就会很慢，正好现在手上有台国内服务器， 可以自建异地组网，中转的速度也能得到保证。

选择

虚拟组网方案很多，支持二层组网的就少了许多，还支持打洞的选择就不多了。

对比了几种方案，最终选择了 N2N / N3N 这类软件。这俩软件差别不大，N3N 是一个活跃贡献者不满 N2N 许可证变更 Fork 出来的 (开源社区日常操作)。

安装

  • Linux
    • 各发行版官方仓库应该都有收录。
    • Arch Linux 用户可以使用我打的包 —— n2n-git 。
  • Windows
    • 使用官方通过 Github Actions 自动构建的包 —— N2N Actions 。
    • 使用我交叉编译的包 —— n2n-win ，启用了更多特性。
  • Android
    • 使用 hin2n.apk ，但已经事实上停更，而且缺了很多功能。
    • 使用我交叉编译的包 —— n2n-droid ，启用了更多特性 (需 ROOT)。

使用

N2N 这软件虽然小众，但网上也有很多使用教程，这里不再赘述。只介绍 Android 在 ROOT 环境下如何使用。

  1. 安装 终端模拟器 。
  2. 将软件和配置放到 终端模拟器 的家目录。
    • Android 10+ 可能还需将 SELinux 设置为 Permissive 模式 —— Behavior changes: apps targeting API 29+  |  Android Developers 。
  3. 在 终端模拟器 的一个 Session 中运行 N2N 。
    • su -c "edge edge.conf -f"。
  4. 在 终端模拟器 另一个 Session 添加路由规则，配置下一跳 IP 地址。
    • ip route add "IP/CIDR" via "IP" dev edge0 table local。
