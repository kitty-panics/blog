'ssl_stapling' ignored, host not found in OCSP responder 'r3.o.lencr.org' in the cert 'fullchain.pem'

Kitty • Panics  included in Nginx
 2021-06-21  304 words   One minute 
Contents

在查看服务器日志时发现 Nginx 中有一个 Warn。

1
2

[warn] 285#285: "ssl_stapling" ignored, host not found in OCSP responder
"r3.o.lencr.org" in the certificate "/XXXX/YYYY/ZZZZ/fullchain.pem"

看到这个警告第一反应是不是 Let’s Encrypt 的服务挂了，毕竟它以前也出过几次问题， 但 PING 了一下发现是正常的，看来得是服务器内部的问题了。

接着排查了 Nginx 配置、DNS、防火墙等居然都没问题。本着 “重启能解决 90% 的问题” 的原则重启试了一下，发现没有报错了。但 “BUG 既不会凭空产生，也不会凭空消失， 它只会从一种形式转化为另一种形式”。

接着经过多次重启测试发现，这个问题是由于 Nginx 在网络未就续时就启动造成的。

对于这种情况，只需编辑 nginx.service 服务文件，添加一行配置即可，目的是让它延迟 2s 启动。

1
2
3
4
5

# 编辑 nginx.service 服务文件
vim /etc/systemd/system/multi-user.target.wants/nginx.service

# 设置延迟 2s 启动
ExecStartPre=/usr/bin/sleep 2
