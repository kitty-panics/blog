2022-05-19 的更新

不知从何时起，IcoMoon 只能上传 SVG 格式的图像了。不过对于简单的 png 格式图片可以用 png2svg 脚本转换成 svg 格式。

 

 

最近有需求要在某软件的状态栏上放一个专属 LOGO，但奈何不支持使用图片。突然想起 Font-Awesome 、Nerd-Fonts 这类项目，用 iconfont 也许是个不错的办法。

了解并实践了一下，作一个总结。

IcoMoon

IcoMoon 应该是目前最简单的方法 (没有之一)，只要有浏览器就行。

  1. 制作出自己想要的 LOGO (png/svg 格式)
  2. 打开网站 -> 点击 “Import Icons” 按钮 (左上角)
  3. 选择刚才制作的 LOGO 图片 -> 点击 “Generate Font” 按钮 (右下角)
  4. 根据自己的需求自定义类名和 Unicode 码位
  5. 点击 “Download” 按钮 (右下角)

就这么简单的点几下，字体就制作好了。

  • 优点：
    • 简便易用
    • 能为每个字符自定义 Unicode 码位
  • 缺点：
    • 如果要制作大字符集字体就很麻烦了

Glyphr Studio / Fonteditor

这两个轻量级的在线字体编辑器也算很简便易用，同样只需浏览器就行。

  1. 打开网站 -> 新建项目
  2. 新建字形 -> 画出/导入自己想要的 LOGO
  3. 导出字体

这两个工具相对于前面的 IcoMoon 稍微专业了些，很适合业余爱好者。

  • 优点：
    • 简便易用
    • 自定义程度更高
  • 缺点：
    • 需要一定制作字体的基础
    • 虽然称得上字体编辑器了，但依然不太适合制作大字符集字体 (浏览器一误关，就啥都没了)

gulp-svgicons2svgfont

一个用 Node.js 写的转换工具，本质上是对 webfonts-generator 包装了一层 CLI， 而 webfonts-generator 本质上又是在调用 svg2ttf, ttf2eot, ttf2woff 等工具 (无限套娃)。

简单的转换示例如下。

1

icon-font-generator "图片目录/*.svg" -o "输出目录"
  • 优点：
    • 能快速批量制作大字符集字体
    • 能为每个字符自定义 Unicode 码位
  • 缺点：
    • 需要安装 Node.js 和涉及命令行操作，可能会是个门槛

也还有其它用 Node.js 写的 svg2font 工具，比如 svgtofont 等， 其本质上也是在调用 svg2ttf, ttf2woff，就不再赘述。

FontCreator / Fontforge

专业的字体制作工具，需要用到它的自然不需要我的赘述，不需要的说了也还是不需要。

  • 优点：
    • 专业的字体制作工具
  • 缺点：
    • 学习成本偏高

#!/usr/bin/env bash

#
# png to svg
#
# 注：
# 需要系统已装 imagemagick, potrace 包
#

# 获取文件名
FILE_NAME=$(echo -e "$1" | sed 's/.png$//g')

# 转换到过渡文件 (PNM 格式)
convert "$FILE_NAME.png" "$FILE_NAME.pnm"
# 转换到 SVG 格式
potrace "$FILE_NAME.pnm" -s -o "$FILE_NAME.svg"

# 删除过渡文件
rm -rf "$FILE_NAME.pnm"
