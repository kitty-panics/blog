部分机型使用 TWRP 后加密手机失败

备用机一直在用 LineageOS 类原生系统，今天突然发现系统居然未加密 (从 Android 7.0 开始默认使用文件级加密 (FBE))，尝试加密还失败了。

开始以为是系统的问题，然后换了一个类原生系统，结果发现还是没能解决问题。

原因

一番排查发现是 TWRP 的问题，是因为 /data 所在的分区 (即 userdata 分区) 占用了加密页脚空间导致的 (在 Device Tree 中的 BoardConfig.mk 文件中定义)。

所以只需缩小 /data 所在的分区 (即 userdata 分区) 为加密页脚留下足够的空间 (16KiB) 即可。

解决

需要先找到 /data 所在的分区

一般来说是 /dev/block/bootdevice/by-name/userdata。

  1. 电脑安装 ADB 工具和手机驱动 (不得不说在 Linux 下使用 ADB 工具 不用安装驱动 真的很棒)
  2. 手机开启 USB 调试模式并连接电脑
  3. 执行 adb logcat > get-logcat.log 命令
  4. 在手机上执行加密操作 (通常是 “设置 -> 安全性和位置信息 -> 加密与凭据 -> 加密手机”)
  5. 等待加密失败并重启
  6. 结束 ADB 操作 (通常是同时按下 “Ctrl 键 + C 键”)
  7. 在 get-logcat.log 中查找含 “E Cryptfs” 字符的行 (示例如下)，并记住这个分区名 (在我的设备上为 /dev/block/bootdevice/by-name/userdata)
1
2

02-26 10:29:27.594   459  7814 E Cryptfs : Bad magic for real block device /dev/block/bootdevice/by-name/userdata
02-26 10:29:27.594   459  7814 E Cryptfs : Orig filesystem overlaps crypto footer region.  Cannot encrypt in place.

再进入 TWRP 执行缩小分区操作

  1. 进入 TWRP，卸载 /data (“Mount” -> “取消标记 Data”)
  2. 手机连接电脑并执行 adb shell 命令 (也可以用 TWRP 内建的终端，但操作起来没电脑舒服)
  3. 执行 ls -l /dev/block/bootdevice/by-name/userdata 命令
    • 此步骤用于显示出 userdata 分区所在的块，记住这个块设备名 (在我的设备上为 /dev/block/mmcblk0p49)
    • 参数 -l 解释: 使用较长格式列出信息
  4. 执行 e2fsck -y -f /dev/block/mmcblk0p49 命令
    • 此步骤用于检查文件系统的完整性
    • 参数 -y 解释: 采取非交动方式，所有的问题均自动以 yes 回答
    • 参数 -f 解释: 即使文件系统没有错误，仍强制检查正确性
  5. 执行下面的命令
    • 此步骤用于过滤出块的大小，并会将这个数值减去 16384 (16) (在我的设备上为 2588025)
    • 参数 -l 解释: 显示文件超级块内容
1

expr $(tune2fs -l /dev/block/mmcblk0p49 | grep "Block count" | awk '{print $3}') - 16384
  1. 执行 resize2fs /dev/block/mmcblk0p49 2588025 命令
    • 此步骤用于调整文件系统的大小

注:

上述中的各分区名、块设备名、块大小只是示例，需根据自己设备的实际情况作相应的修改。

最后重新加密

再重新执行加密操作，不出意外应该能正常执行加密了。

扩展阅读

一键命令

如果您已知晓原理，并觉得上述操作繁锁的话，可以使用下面的一键命令。

1

resize2fs /dev/block/mmcblk0p49 `expr $(tune2fs -l /dev/block/mmcblk0p49 | grep "Block count" | awk '{print $3}') - 16384`

Please run 'e2fsck -f /dev/block/mmcblk0p49' first.

可能还会遇到一个错误提示 Please run 'e2fsck -f /dev/block/mmcblk0p49' first.， 但是运行了 e2fsck 后还是无法加密。

对此，只需对 resize2fs 命令加上 -f 参数即可，表示强制调整大小。
