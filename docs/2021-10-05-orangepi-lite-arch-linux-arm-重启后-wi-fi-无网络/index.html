OrangePi-Lite (Arch Linux ARM) 重启后 Wi-Fi 无网络

Kitty • Panics  included in Linux
 2021-10-05  308 words   One minute 
Contents
  • Debug
  • 解决

上一篇说到在 OrangePi Lite 中安装了 Arch Linux ARM ，由于 Orange Pi Lite 没有有线网络接口， 所以只能使用 Wi-Fi。Wi-Fi 连接方案如下。

  1. 使用 wpa_supplicant 连接到 Wi-Fi。
  2. 使用 dhcpcd 请求分配 IP 地址。

在使用中遇到一个问题，每次重启后随缘没有网络连接，但在路由器后台又能看见 Orange Pi Lite 设备。

Debug

开发板通过 “USB 转 TTL” 模块与电脑相连，再次启动观察启动日志，发现 wpa_supplicant 服务耗费了大量时间，但具体时间不定，而 dhcpcd 服务一直在等待它，可惜的是 dhcpcd 服务不会一直等待它。

因为 dhcpcd 默认超时时间只有 30s。

解决

找到问题就好解决了，只需编辑 dhdpcd 的 systemd.unit ，将等待时间设置为 -t 0 (无限) 即可。

完整的 dhcpcd@接口.service 如下。

[Unit]
Description=dhcpcd on %I
Wants=network.target
Before=network.target
BindsTo=sys-subsystem-net-devices-%i.device
After=sys-subsystem-net-devices-%i.device

[Service]
Type=forking
PIDFile=/run/dhcpcd/%I.pid
ExecStart=/usr/bin/dhcpcd -q -w -t 0 %I
ExecStop=/usr/bin/dhcpcd -x %I
Restart=always

[Install]
WantedBy=multi-user.target
