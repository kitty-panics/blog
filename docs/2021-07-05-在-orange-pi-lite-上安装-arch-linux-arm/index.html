在 Orange Pi Lite 上安装 Arch Linux ARM

前段时间将云服务器的系统都换成了 Arch Linux，最近意外发现角落吃灰的 Orange Pi Lite。

正好手上有张多余的 MicroSD 卡，烧录了几个系统发现都不如意 (官方提供的系统太很老了， 而且还是用 Armbian 改的，而 Armbian 又基于 Debian，但我不喜欢 Debian)。索性也换成 Arch Linux (ARM) 吧。

不过 Arch Linux ARM 官方并不支持 Orange Pi，所以无法用 dd 烧录系统。以下是手动安装步骤。

设备准备篇

  1. 将 MicroSD 卡开头清零
    • sudo dd if=/dev/zero of=/dev/sdX bs=1M count=8
  2. 对 MicroSD 卡进行分区 (磁盘标签类型为 dos)
    • sudo cfdisk /dev/sdX
  3. 将分区格式化成 Ext4
    • sudo mkfs.ext4 /dev/sdX1
  4. 挂载分区至 /mnt (随便挂载到哪，这只是一种约定俗成)
    • sudo mkdir /mnt
    • sudo mount /dev/sdX1 /mnt

系统安装篇

  1. 下载 Arch Linux ARM 根文件系统
    • curl -LO "https://archlinuxarm.org/os/ArchLinuxARM-armv7-latest.tar.gz"
  2. 解压根文件系统至挂载目录
    • sudo bsdtar -xpf ArchLinuxARM-armv7-latest.tar.gz -C /mnt/
  3. 编译并复制 boot-script 镜像至挂载目录
    • sudo cp -rf boot.scr /mnt/boot/
  4. 编译并写入 u-boot 镜像至 MicroSD 卡
    • sudo dd if=u-boot-sunxi-with-spl.bin of=/dev/sdX bs=1024 seek=8
  5. 卸载分区并弹出 MicroSD 卡
    • sudo umount /mnt
    • sudo eject /dev/sdX

注：

Arch Linux ARM 根文件系统默认不提供 boot-script 和 U-BOOT 引导程序。 如果你不具备编译环境或不会编译，可以使用我已编译好的文件。

系统体验篇

  1. 初始化 pacman Keyring
    • pacman-key --init
  2. 填充 Arch Linux ARM 软件包的签名密钥
    • pacman-key --populate archlinuxarm

注：

Arch Linux ARM 官方固件包中不提供 OrangePi Lite 的 Wi-Fi 驱动，如果你不具备编译环境或不会编译， 可以从下面下在我编译好的驱动软件包。

  • kitty-panics/OrangePi-Lite-RTL8189FTV

扩展阅读

Arch Linux ARM 根文件系统默认有两个用户，其登录密码与用户名相同。

一个是 root 管理员用户，另一个是 alarm 普通用户 (Arch Linux ARM 的缩写)。
