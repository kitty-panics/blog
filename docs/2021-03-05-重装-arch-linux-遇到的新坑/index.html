ArchWiki 里更新了 GRUB2 的 LUKS2 配置文档。

2021-06-08 的更新

世界变化真快，GRUB2 也支持 LUKS2 了 (详见：GRUB 2.06 Released With BootHole Fixes, LUKS2 Encrypted Volume Support )。

但是支持的还不完善，需要手动干预才能正常使用，操作方法详见 AUR - grub-git 的评论区。

 

 

之前的电脑坏掉了，顺便大力出奇迹，把硬盘也搞炸了，手上的备份还是一年多前的。 没办法新电脑只能选择重装了，顺便说说这次重装遇到的新坑。

base 包组

在 2019 年 10 月，base 包组被替换为同名的元包，并且元包不再带内核、编辑器、 文件系统、man-pages 等等。所以新装系统时，原来一句 pacstrap /mnt base 能搞定的事， 变成了 pacstrap /mnt base 内核包 文件系统包 编辑器包。

其中内核、编辑器、文件系统又有多种选择，无疑是有选择困难症的新入门的人增加了些难度。

内核 编辑器 文件系统
linux nano e2fsprogs
linux-lts vi xfsprogs
linux-zen vim btrfs-progs
linux-hardened neovim ……

注：

所以基于这一点，2019-10 以前的 Arch Linux 安装教程都可以不用看了。

LUKS1/2

在 Linux 上全盘加密磁盘，一般使用的是 cryptsetup 这个 dm-crypt 用户空间工具。在 v2.1.0 版本之前，--type luks 默认等同于 --type luks1，如果要使用 LUKS2 需指明 --type luks2， 这么多年下来，这几乎是一种约定俗成了。

但在 v2.1.0 版本之后完全反过来了，--type luks 默认等同于 --type luks2，要使用 LUKS1 需指明 --type luks1。

如果只是普通的加密磁盘这一变动并不会导致什么问题，但如果是加密 BOOT 分区，按照之前约定俗成使用 --type luks 就会导致系统无法启动。因为 Bootloader 只有 GRUB2 支持 LUKS，而 GRUB2 又只支持 LUKS1。
