不需要证书签名请求

依次执行下面两条命令即可。

  1. 生成私钥
    • RSA
      • openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:2048 -out 私钥.key
    • ECDSA
      • openssl genpkey -algorithm EC -pkeyopt ec_paramgen_curve:P-256 -out 私钥.key
  2. 生成证书
    • openssl req -x509 -days 90 -key 私钥.key -out 证书.crt

上面两条命令可以合并成一条命令。如果要对输出密钥进行加密，去掉最后面的 -nodes 参数即可。

1

4
5

# 一步生成 RSA 私钥 + 证书
openssl req -x509 -newkey rsa:2048 -days 90 -keyout 私钥.key -out 证书.crt -nodes

# 一步生成 ECDSA 私钥 + 证书
openssl req -x509 -newkey ec -pkeyopt ec_paramgen_curve:prime256v1 -days 90 -keyout 私钥.key -out 证书.crt -nodes

需要证书签名请求

依次执行下面三条命令即可。

  1. 生成私钥
    • RSA
      • openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:2048 -out 私钥.key
    • ECDSA
      • openssl genpkey -algorithm EC -pkeyopt ec_paramgen_curve:P-256 -out 私钥.key
  2. 生成证书签名请求
    • openssl req -new -sha256 -key 私钥.key -out 证书签名请求.csr
  3. 生成证书
    • openssl x509 -req -days 90 -in 证书签名请求.csr -signkey 私钥.key -out 证书.crt

上面第 1 和第 2 步也可以用一条命令实现。如果要对输出密钥进行加密，去掉最后面的 -nodes 参数即可。

1
2
3
4
5

# 一步生成 RSA 私钥 + 证书签名请求
openssl req -new -newkey rsa:2048 -keyout 私钥.key -out 证书签名请求.csr -nodes

# 一步生成 ECDSA 私钥 + 证书签名请求
openssl req -new -newkey ec -pkeyopt ec_paramgen_curve:prime256v1 -keyout 私钥.key -out 证书签名请求.csr -nodes

扩展阅读

私钥加密

如果要对私钥进行加密，在私钥生成命令中加上 -aes-256-cbc -pass pass:您的密码 参数即可。

非交互式输入 Subject

在生成证书过程中会让用户输入一些信息，如果不想交互式输入，可以使用 -subj 参数。

1

-subj "/C=CN/ST=Guangdong/L=Shenzhen/O=Tencent/OU=Social Network Group/CN=qcloud.com/emailAddress=sb@qq.com"

以下是 -subj 参数的详细说明。

缩写 全名 中文解释 示例
C Country Name 国家名称代码 (两位代码) CN
ST State or Province Name 州或省名称 (全名) Guangdong
L Locality Name 城市名称 Shenzhen
O Organization Name 组织名称 (公司) Tencent
OU Organizational Unit Name 组织单位名称 (部门) Social Network Group
CN Common Name 通用名称 (域名或您的姓名) qcloud.com
emailAddress Email Address 邮件地址 sb@qq.com

curl 访问使用自签证书的网站

如果用 curl 命令访问使用了自签证书的网站，会有以下提示。

curl: (60) SSL: no alternative certificate subject name matches target host name
'https://使用自签证书的网站.com' More details here: https://curl.se/docs/sslcerts.html

curl failed to verify the legitimacy of the server and therefore could not
establish a secure connection to it. To learn more about this situation and
how to fix it, please visit the web page mentioned above.

解决方法很简单，加上 -k 参数即可。
