GRUB2 救援模式下解密 LUKS1

Kitty • Panics  included in GRUB2
 2018-10-12  242 words   One minute 
Contents
  • 操作步骤

众所周知，GRUB2 有两种模式，一种是正常模式，支持超多的特性、超多的模块，使得 GRUB 有着 GRUB OS 的称号。另一种就是救援模式，只支持最基本的命令，最基本命令的功能还是残缺的， 但也足够保证进入到正常模式。

如果 BOOT 分区有加密 (LUKS1)，GRUB2 在启动时会先解密 BOOT 分区。但是当密码不小心输错了， 并不会让你重新输入密码，而是直接进入救援模式。

操作步骤

那么就需要在救援模式中使用极其有限的命令重新解密并进入正常模式。

  1. 重新输入密码解密 BOOT 分区 (可通过 ls 命令查看设备名)
    • cryptomount 设备名
  2. 插入 normal 模块
    • insmod normal
  3. 进入 normal 模式
    • normal
