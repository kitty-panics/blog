复现

在 大量的频繁的 读写时可复现。

原因

UAS (USB Attached SCSI) 是 USB 3.0 的一种新的传输协议 (支持多线程，命令排队等)， 默认启用。当然这么厉害的东西也不是轻易就能使用的，得满足以下条件。

  1. Device hardware supporting UAS.
  2. Device firmware supporting UAS.
  3. UAS support in the USB driver stack.
  4. UAS-compatible system controller.

四者缺一不可，否则就会出现上述的问题。

解决

可通过添加一个内核参数禁用掉 UAS 来解决此问题。

使用内核命令行

1

usb-storage.quirks=Vendor_ID:Product_ID:u

使用 modprobe

在 /etc/modprobe.d 目录中，创建一个 uas.conf 文件， 其内容如下。

1

options usb-storage quirks=Vendor_ID:Product_ID:u

其中 Vendor_ID 和 Product_ID 可通过 lsusb 命令获取。
