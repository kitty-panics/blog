[ TIME ] Timed out waiting for device /dev/disk/by-uuid/XXXX-XXXX
[DEPEND] Dependency failed for /boot/efi.

[DEPEND] Dependency failed for File System Check on /dev/disk/by-uuid/XXXX-XXXX.

(or press Control-D to continue):

虽然按下 Control-D 后能正常启动，但看着也烦。对于这个问题，网上有多种解决方案。

  1. /etc/fstab 中的磁盘的 UUID 配置错误。
  2. /etc/mkinitcpio.conf 中没有添加 VFAT 模块。

当然，也有人和我一样，这些解决方案并没有用。其实还可能存在一个特别小众的情况。

关机时间过长

众所周知，Linux 系统在关机时，会向正在运行的进程发送终止信号 (sigterm)，但有些进程并不会处理这个信号， 而是继续运行。这时 systemd 会等待它一段时间 (默认是 90 秒)，再强行 kill 掉，但这会导致关机时间过长。

于是有人就会修改 /etc/systemd/system.conf 配置文件中的 DefaultTimeoutStopSec 设置项，将等待时间缩短。

误配置 systemd

但是，有些教程或人 (比如我) 会将 DefaultTimeoutStartSec 设置项也一并缩短，一般来说不会这不会有什么问题。

但如果磁盘启用了 LUKS 加密，而解密时间太长，超过了 DefaultTimeoutStartSec 的时间，就 Timed out 了。

因此，只需将 DefaultTimeoutStartSec 设置项还原即可。
