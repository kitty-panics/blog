R7000 2020 在 Linux 中完全关闭 N 卡

Kitty • Panics  included in Linux
 2021-09-24  497 words   One minute 
Contents
  • bbswitch
  • acpi_call

在 Linux 中完全关闭 NVIDIA 独立显卡就三个方法。

  1. 在 BIOS 中禁用。
  2. 使用 bbswitch 。
  3. 使用 acpi_call 。

第 1 种方法应该是最简便最彻底的，可惜绝大多数笔记本的 BIOS 并不支持禁用独显。 那就只能在第 2、3 种方法中根据设备情况二选一了。

如果仅仅是实现标题中的需求，对于绝大多数笔记本来说 bbswitch 和 acpi_call 都行， 不过在 R7000 上用 bbswitch 会导致一些新的问题。

bbswitch

bbswitch 的 master 分支最后一次更新是在 2013 年，各发行版也是打包的此版本。 而那时的 bbswitch 仅支持 Intel + NVIDIA 组合。而 R7000 是 AMD + NVIDIA 组合， 就得自行编译 develop 分支了 (在 Arch Linux 上，可以使用 bbswitch-ati-git 这个 AUR 包)。

bbswitch 使用教程一大堆，这里就不再赘述，当然也不建议在 R7000 上使用它， 因为使用 bbswitch 虽然能关闭 NVIDIA 卡，但会导致风扇以最大速度运行。

  • 此 Issues：Asus UX501VW: Disabling GPU causes loss of fan control (with fan running at max speed) 。

而按照 Issues 中给出的方法添加内核参数后，并不能解决问题，还会导致触摸板无法使用。

acpi_call

acpi_call 最后一次更新也是在 2013 年，不过还能正常使用。唯一的缺点就是 turn_off_gpu.sh 中只包含一小部分 GPU 总线信息，可能会有人运行了下发现无效就放弃了，其实只需要给出正确的 GPU 总线信息就能使用了，R7000 2020 的 GPU 总线信息为 \_SB.PCI0.GPP0.PG00。

要在启动时关闭 NVIDIA 卡只需在 /etc/tmpfiles.d 目录中创建一个 nvidia.conf 文件， 其内容如下。

1

w /proc/acpi/call - - - - \\_SB.PCI0.GPP0.PG00._OFF
