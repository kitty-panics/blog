Linux 版的存储空间隔离

用过 Android 系统的同学都知道，几乎所有软件都会请求存储权限，然后在里面各种乱拉屎， 导致存储空间乱七八遭的，每次找个东西得翻动半天。为此有开发者写了个 存储空间隔离 的工具，能让存储空间重回整洁，缺点就是需要 ROOT 权限和收费。

Linux 系统上也是如此，大量软件不遵守 XDG 规范 ，导致家目录一团糟，好在 Linux 上可以轻松实现 “存储空间隔离” 这一功能，而且还是免费的。

这里需要用到一款沙盒软件 —— Firejail (也可以选择其它 沙盒软件 ，只是我更偏爱 Firejail)。

使用

1. 为需要隔离的软件写一个 Wrapper

Wrapper 放到自己喜欢的目录 (我的是 $HOME/.local/Redirect-Storage-Bin)，重要的是将此目录置于 PATH 环境变量开头。这样在运行需要隔离的软件时，除非指明 /usr/bin/ 绝对路径，会优先使用 Wrapper。

以 Steam 为例。通过此 Wrapper 运行 Steam 时，Steam 会将 “~/.local/Redirect-Storage-Data/steam” 当作家目录。

#!/usr/bin/env bash

#
# steam (PKG: steam)
#

# 创建隔离目录
mkdir -p "$HOME/.local/Redirect-Storage-Data/steam"
# 启动 Steam
exec _firejail --dns=223.6.6.6 --private="$HOME/.local/Redirect-Storage-Data/steam" /usr/lib/steam/bin_steam.sh "$@"

上面的 _firejail 命令是 Firejail 的 Wrapper。这样做能在当 Wrapper 数量很多时，可以快速修改 Firejail 参数。

2. 定制 _firejail 参数

可根据需要定制 Firejail 的参数，比如禁止读取 /media 媒体设备挂载目录、启用 Apparmor……等。

#!/usr/bin/env bash

#
# Firejail Wrapper
#

exec /usr/bin/firejail \
    --apparmor \
    --blacklist="/media" \
    --blacklist="/mnt" \
    --blacklist="/swap.file" \
    --caps.drop=all \
    --disable-mnt \
    --nodvd \
    --nogroups \
    --nonewprivs \
    --noprofile \
    --noroot \
    --notv \
    --nou2f \
    --quiet "$@"
    #--memory-deny-write-execute \
    #--private-dev \
    #--private-tmp \

3. 链接到 XDG User Directories

为方便查找隔离空间内的文件，可以将 Redirect-Storage-Data 目录链接到 XDG User Directories 。

  • XDG_DESKTOP_DIR: ln -s $HOME/.local/Redirect-Storage-Data $HOME/Desktop/Redirect-Storage-Data
  • XDG_DOCUMENTS_DIR: ln -s $HOME/.local/Redirect-Storage-Data $HOME/Documents/Redirect-Storage-Data
  • XDG_DOWNLOAD_DIR: ln -s $HOME/.local/Redirect-Storage-Data $HOME/Downloads/Redirect-Storage-Data
  • XDG_MUSIC_DIR: ln -s $HOME/.local/Redirect-Storage-Data $HOME/Music/Redirect-Storage-Data
  • XDG_PICTURES_DIR: ln -s $HOME/.local/Redirect-Storage-Data $HOME/Pictures/Redirect-Storage-Data
  • XDG_PUBLICSHARE_DIR: ln -s $HOME/.local/Redirect-Storage-Data $HOME/Public/Redirect-Storage-Data
  • XDG_TEMPLATES_DIR: ln -s $HOME/.local/Redirect-Storage-Data $HOME/Templates/Redirect-Storage-Data
  • XDG_VIDEOS_DIR: ln -s $HOME/.local/Redirect-Storage-Data $HOME/Videos/Redirect-Storage-Data
