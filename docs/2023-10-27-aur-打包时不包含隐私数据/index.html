AUR 打包时不包含隐私数据

Kitty • Panics  included in Linux
 2023-10-27  266 words   One minute 
Contents
  • 步骤

这两天拆包时瞥了一眼元数据文件 .BUILDINFO，突然发现其中的 builddir、startdir、 installed 变量存在隐私泄漏的风险。

去翻了下 pacman 的 Commits，原来早在五六年前就加进来了。

  • makepkg: add the $startdir to package .BUILDINFO (0428f621)
  • Append architecture information to ‘installed’ elements in .BUILDINFO (5698d7b6)

installed 在复现 BUG 时还能发挥一定的作用，builddir 和 startdir 就完全没作用了， 将它们全部移除。

步骤

将其移除也很简单，编辑 /usr/bin/makepkg 命令，将下面的内容注释掉即可。

	write_kv_pair "builddir"  "${BUILDDIR}"
	write_kv_pair "startdir"  "${startdir}"

    # ………………

	local pkginfos_parsed=($(LC_ALL=C run_pacman -Qi | awk -F': ' '\
		/^Name .*/ {printf "%s", $2} \
		/^Version .*/ {printf "-%s", $2} \
		/^Architecture .*/ {print "-"$2} \
		'))
	write_kv_pair "installed" "${pkginfos_parsed[@]}"

为防止 pacman 在更新后覆盖掉修改，可将其属性设置成不可修改。
