2022-02-04 的更新

截止今日，v11.0.0 可通过 flameshot gui 命令实现截图后自动退出，本文可废弃。

 

 

截图工具比较喜欢 FlameShot ，截图后有编辑功能，还算比较轻量，UI 也很好看。 不过用了几次后，发现它有一个逼死强迫症的问题。

不能设置为每次截图后自动退出，它会驻留在后台。

因为对我来说截图并不一个常用的功能，每次截图后，还占着一个托盘位置看着实在碍眼。

分析

本来打算根据程序返回的状态码来判断是否截图成功，结果 FlameShot 不管什么情况都返回 0。

逛了一圈 Issues，发现有人提出过 这个问题 ，也在一个 PR 中被修复了。 不过不知为啥已经是最新版的 FlameShot 依然有这问题，好在因为这个 PR 意外了发现 --raw 参数有妙用。

实现

然后写了个 Shell 脚本以不优雅的方式解决了这个问题。

#!/usr/bin/env bash

#
# FlameShot 截图后结束进程
#

# 直接截取整个桌面
Entire_Desktop() {
    # 开始截图
    flameshot full --raw --path "$HOME/Pictures/"
    # 等待 2 秒
    sleep 2s
    # 结束 FlameShot
    killall flameshot
}

# 手动框选截图
Manual_Capture() {
    # 开始截图
    flameshot gui --raw --path "$HOME/Pictures/"
    # 等待 2 秒
    sleep 2s
    # 结束 FlameShot
    killall flameshot
}

# 帮助页
Manual_Page() {
    cat << EOF
    Usage: _flameshot [options] [arguments]

    Options:
    -h, --help     Displays this help
    -v, --version  Displays version information

    Arguments:
    full           Capture the entire desktop.
    select         Start a manual capture in GUI mode.
EOF
}

# 版本信息
Show_Version() {
    cat << EOF
    _flameshot v0.0.3

    It is written in the scripting language Bash.
EOF
}

# 参数判断
case "$1" in
    "h" | "-h" | "help" | "--help")
        Manual_Page
        ;;
    "v" | "-v" | "version" | "--version")
        Show_Version
        ;;
    "f" | "-f" | "full" | "--full")
        Entire_Desktop
        ;;
    "s" | "-s" | "select" | "--select")
        Manual_Capture
        ;;
    *)
        cat << EOF
    No options/arguments were used or the wrong options/arguments were used.

    Use the -h/-help option for detailed help.
EOF
        ;;
esac

释疑

截图后为何要等待 2 秒

为了让 FlameShot 截图后有时间写入文件到磁盘 (2 秒已足够)。

为何要加 --raw 参数

如果不加 --raw 参数，脚本的第 10/20 行中的动作几乎是一瞬间执行完成，然后就会继续执行 kill 操作 (尽管 FlameShot 还在后台运行)。这样的话就不能在截图后使用编辑功能了，因为只有 2 秒的时间来编辑。

而加了这个参数，FlameShot 就会一直处于前台，直到完成截图或取消截图才执行下一步 kill 操作。

扩展阅读

被 Kill 掉后截的图不能粘贴

是这样的，这是 X 的 剪切板 机制所决定的。鱼和熊掌不可兼得。

键盘流

上面的脚本肯定是不能手动运行的，太低效了，应该以 _flameshot 名称保存到 PATH 中，然后设置快捷键调用。

我使用的是 i3wm ，所以就以此为例。

bindsym $mod+m mode "+Multimedia"
mode "+Multimedia" {
    bindsym f exec --no-startup-id _flameshot full         ,mode "default"
    bindsym Shift+f exec --no-startup-id _flameshot select ,mode "default"
    bindsym Escape mode "default"
    bindsym Return mode "default"
    bindsym $mod+m mode "default"
}

按下 Win+M键 后，再按下 F键 可进行全屏截图。

按下 Win+M键 后，再按下 Shift+F键 可手动选择区域进行截图。
