2022-06-01 的更新

家里升级到了 1000M 宽带，换了一个新光猫。登录与重启相对于之前有大两个变化。

  1. 登录密码无需经过 Base64 编码。
  2. 重启不再需要 CurrentLocalTime (当前时间) 参数。

 

 

我这里的移动宽带每隔一段时间会重新分配一个 IP，然后用这个 IP 会有些网站无法访问。 好在重启光猫后能获得一个新的 IP。

# -*- coding: UTF-8 -*-

#
# 重启移动光猫
#

# 导包
import re
import requests

# 定义用户名、密码
USER_NAME = "XXXXXXXX"
PASSWORD = "YYYYYYYY"

# 定义 "User Agent"
USER_AGENT = "Mozilla/5.0 (Windows NT 6.1; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/90.0.4430.93 Safari/537.36"
HEADER = {"User-Agent": USER_AGENT}
# 定义 "登录/重启 URL"
LOGIN_URL = "http://192.168.1.1"
REBOOT_URL = "http://192.168.1.1/web/cmcc/gch/getpage.gch?pid=1002&nextpage=web/cmcc/gch/iot_advance_setting_t.gch"

# 创建 Session
login_session = requests.session()

# 登录光猫
def login_Cat():
    # 获取 HTML 内容
    get_html = login_session.get(LOGIN_URL)
    # 解析出 Frm_Logintoken 的值
    get_frm_logintoken = re.search(r'getObj\("Frm_Logintoken"\)\.value = "([0-9])";', get_html.text).group(1)

    # 定义登录参数
    login_data = {
        'frashnum': '',
        'action': 'login',
        'Frm_Logintoken': get_frm_logintoken,
        'username': USER_NAME,
        'logincode': PASSWORD,
        'textpwd': '',
        'ieversion': '1'
    }
    # 登录
    login_session.post(LOGIN_URL, headers=HEADER, data=login_data, allow_redirects=True)


# 重启光猫
def reboot_Cat():
    # 获取 HTML 内容
    get_html = login_session.get(REBOOT_URL)
    # 解析出 IF_ERRORTYPE 的值
    get_if_errortype = re.search(r'(\\x2d)([0-9].*[0-9])', get_html.text).group(2)
    # 解析出 _SESSION_TOKEN_USER 的值
    get__session_token_user = re.search(r'var session_token = "(.*)";', get_html.text).group(1)

    # 定义登录参数
    reboot_data = {
        'IF_ACTION': 'devrestart',
        'IF_ERRORSTR': 'SUCC',
        'IF_ERRORPARAM': 'SUCC',
        'IF_ERRORTYPE': "-" + get_if_errortype,
        'flag': '1',
        '_SESSION_TOKEN_USER': get__session_token_user
    }
    # 重启
    login_session.post(REBOOT_URL, headers=HEADER, data=reboot_data, allow_redirects=True)


# 登录与重启
login_Cat()
reboot_Cat()

# 关闭 Session
login_session.close()

扩展阅读

Base64 编码很常用，如果要在 Linux 系统下用 echo 'String' | base64 命令进行 Base64 编码， 需要为 echo 命令加上 -n 参数。

因为 echo 命令默认会在行尾追加一个换行符，会导致编码结果不正确。
